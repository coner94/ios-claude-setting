## When to Activate

- 코드 리뷰가 필요하거나 요청받았을 때
- 사용자가 `/code-review`로 직접 호출할 때

## Step 1: 변경 범위 파악

- 변경된 파일 목록과 diff를 분석한다
- 변경의 목적과 의도를 파악한다
- 영향받는 레이어와 모듈을 확인한다

## Step 2: 코드 분석

각 변경 사항을 아래 관점에서 검토한다.

### 정확성

자동화 도구가 잡을 수 없는 로직 오류를 중점적으로 본다.

- 코드가 실제로 요구사항을 구현하는가 — 이름은 맞는데 동작이 다른 경우를 확인한다
- 모든 분기(if/switch/guard)가 의도된 결과를 반환하는가 — 누락된 케이스가 없는가
- 비동기 흐름에서 결과 순서에 의존하는 코드가 없는가
- 상태 변화가 예측 가능한 순서로 일어나는가
- 해피패스 외에 경계값(nil, 빈 컬렉션, 최대값), 실패 흐름(네트워크 오류, 타임아웃), 비정상 사용 패턴(중복 요청, 빠른 인터랙션)이 구현에서 처리되는가

### 안전성

설계 수준의 안전성을 본다.

- 사용자 입력 또는 외부 데이터가 사용 전에 검증되는가
- 에러가 적절한 레이어에서 처리되는가 — ViewModel이 처리해야 할 에러를 View가 삼키고 있진 않은가
- 예외 상황(네트워크 실패, 빈 응답, 타임아웃)에서 크래시 없이 복구되는가

### 단순성

처음 보는 사람 기준으로 판단한다.

- 이 코드를 처음 보는 팀원이 5분 안에 의도를 파악할 수 있는가
- 동일한 로직이 두 곳 이상에 있는가 — 있으면 공통화를 제안한다
- 단 한 곳에서만 쓰이는 프로토콜, 제네릭, 추상 계층이 없는가
- 함수가 30줄을 넘는가 — 넘으면 분리 가능한지 검토한다
- 변수명/함수명이 구현 방식이 아닌 의도를 드러내는가

### 성능

설계 수준에서 발생하는 성능 문제를 본다.

- 이 변경이 불필요한 전체 리로드나 전체 재렌더링을 트리거하지 않는가
- 기존 캐싱이나 최적화 로직을 우회하거나 무력화하지 않는가
- 데이터나 상태가 필요한 범위를 넘어 과도하게 전파되지 않는가

### 컨벤션

- SwiftLint 또는 `.claude/rules/swift-style.md` 규칙을 위반하는 코드가 없는가

### 테스트

테스트가 실제로 의미 있는지를 본다.

- 핵심 비즈니스 로직에 테스트가 존재하는가
- 테스트가 구현 세부사항이 아닌 외부 동작을 검증하는가
- 테스트 이름과 구조만 봐도 무엇을 검증하는지 알 수 있는가
- Mock이 실제 의존성의 동작을 충분히 대변하는가

**엣지케이스 / 예외 커버리지**: 해피패스 테스트만 있는지 반드시 확인한다.

- nil, 빈 값, 경계값에 대한 테스트가 있는가
- 실패 흐름(에러, 타임아웃, 빈 응답)에 대한 테스트가 있는가
- 예상치 못한 입력(잘못된 형식, 범위 초과)에 대한 테스트가 있는가
- 해피패스 테스트 수 대비 예외/실패 테스트 수가 현저히 적으면 보완을 요청한다

## Step 3: 리뷰 결과 작성

```markdown
# Code Review: {PR 제목 또는 변경 요약}

## 요약
- 전체적인 코드 품질 평가 (1~2줄)
- 승인 여부: Approve / Request Changes / Comment

## Critical (반드시 수정)
- [{파일명}:{라인}] {문제 설명}
  - 원인: {왜 문제인지}
  - 제안: {구체적 수정 방안}

## Warning (수정 권장)
- [{파일명}:{라인}] {문제 설명}
  - 제안: {개선 방안}

## Suggestion (선택적 개선)
- [{파일명}:{라인}] {제안 내용}

## 긍정적 피드백
- {잘 작성된 부분에 대한 코멘트}
```

---

## Rules

- **코드를 직접 수정하지 않는다.** 리뷰 의견만 제시한다.
- 심각도를 Critical / Warning / Suggestion으로 명확히 구분한다.
- 문제 지적 시 반드시 구체적인 파일명과 라인을 명시한다.
- 단순 스타일 지적보다 로직/안전성/단순성 문제를 우선한다.
- 잘 작성된 코드에 대한 긍정적 피드백도 포함한다.
- Swift 6 Concurrency 관련 데이터 레이스 위험을 식별한다.
