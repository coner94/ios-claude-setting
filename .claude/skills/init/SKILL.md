# 프로젝트 초기화 — CLAUDE.md 생성

## 실행 흐름

### 1단계: 기존 프로젝트 여부 판단

아래 파일이 하나라도 존재하면 **기존 프로젝트**로 판단한다:
- `*.xcodeproj`, `*.xcworkspace`
- `Package.swift`, `Podfile`, `Cartfile`
- `*.swift` 소스 파일
- `package.json`, `build.gradle`, `Cargo.toml` 등 빌드 설정 파일

존재하지 않으면 **신규 프로젝트**로 진행한다.

---

### 2단계-A: 기존 프로젝트 분석

다음 항목을 자동으로 분석한다:

1. **프로젝트 이름**: 루트 디렉토리명, xcodeproj명, package.json 등에서 추출
2. **기술 스택**: 언어, 프레임워크, 의존성 매니저 (SPM/CocoaPods/npm 등)
3. **최소 지원 버전**: 프로젝트 설정에서 deployment target 확인
4. **아키텍처 패턴**: 폴더 구조와 코드 패턴으로 추론 (MVVM, MVC, Clean Architecture 등)
5. **폴더 구조**: 주요 디렉토리와 역할 정리
6. **빌드 & 실행 방법**: 빌드 명령어, 스킴, 타겟 확인
7. **테스트 구조**: 테스트 타겟, 테스트 프레임워크

분석 결과를 사용자에게 요약해서 보여주고, 보완이 필요한 부분을 질문한다:
- "프로젝트의 목적/개요를 한 줄로 설명해주세요"
- "분석 결과 중 수정하거나 추가할 내용이 있나요?"

---

### 2단계-B: 신규 프로젝트 대화형 수집

아래 질문을 **하나씩 순서대로** 진행한다. 한 번에 모든 질문을 던지지 않는다.

**필수 질문:**

1. "프로젝트 이름이 뭔가요?"
2. "어떤 프로젝트인가요? (간단한 설명)"
3. "사용할 언어와 프레임워크는 뭔가요?" (예: Swift + UIKit, Swift + SwiftUI, React, Flutter 등)
4. "아키텍처 패턴이 정해져 있나요?" (예: MVVM, Clean Architecture, 미정 등)

**선택 질문 (해당 시):**

5. "최소 지원 버전이 있나요?" (예: iOS 16.0)
6. "의존성 매니저는 뭘 쓸 예정인가요?" (예: SPM, CocoaPods)
7. "그 외 Claude가 알아야 할 컨벤션이나 규칙이 있나요?"

사용자가 "없음", "스킵", "패스" 등으로 답하면 해당 항목은 생략한다.

---

### 3단계: CLAUDE.md 생성

수집한 정보를 바탕으로 프로젝트 루트에 `CLAUDE.md`를 생성한다.

**생성 형식:**

```markdown
# {프로젝트 이름}

## 개요
{프로젝트 설명}

## 기술 스택
- 언어: {언어}
- 프레임워크: {프레임워크}
- 최소 지원 버전: {버전}
- 의존성 매니저: {매니저}

## 아키텍처
{아키텍처 패턴 및 레이어 설명}

## 폴더 구조
{주요 디렉토리와 역할}

## 빌드 & 실행
{빌드 명령어, 실행 방법}

## 컨벤션
- .claude/rules/ 하위 규칙 파일 참조
{추가 컨벤션}
```

---

### 4단계: 확인

- 생성된 CLAUDE.md 내용을 사용자에게 보여준다
- 수정 요청이 있으면 반영한다
- CLAUDE.md가 이미 존재하면 덮어쓰기 전에 반드시 확인한다

## 주의사항

- 정보가 부족한 섹션은 억지로 채우지 말고 생략한다
- 분석으로 확실하지 않은 내용은 추측하지 말고 사용자에게 묻는다
- CLAUDE.md는 200줄 이내로 간결하게 유지한다
