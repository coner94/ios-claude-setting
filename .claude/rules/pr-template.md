# PR 작성 규칙

### 제목
```
[티켓번호] 간결한 설명 (70자 이내)
```
- 티켓번호: Jira, Monday, GitHub Issue 등 업무 툴의 이슈 키
- 예: `[PROJ-123] 소셜 로그인 화면 추가`
- 티켓이 없는 경우: `[NO-TICKET] 오타 수정`

### 필수 섹션
- **개요**: 변경 사항 1~2줄 요약
- **변경 내용**: 구체적인 변경 사항 목록
- **테스트**: 테스트 체크리스트

### 선택 섹션 (해당 시에만)
- **다이어그램**: 복잡한 로직 변경 시 Mermaid로 시각화
- **스크린샷**: UI 변경 시 Before/After
- **참고 사항**: 리뷰어가 알아야 할 컨텍스트

### PR 크기
- **300줄 이하** 권장 (리뷰 파일 기준)
- 초과 시 PR 분리 검토
- 자동 생성 파일(xib, storyboard, 패키지)은 제외
